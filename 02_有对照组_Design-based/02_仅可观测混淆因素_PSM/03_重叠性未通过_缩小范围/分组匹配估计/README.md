# 分组匹配估计

## 适用条件

- 重叠性检查整体未通过，但**存在明确的业务分组**
- 某些分组内部的重叠性良好，可以在该分组内进行有效匹配

## 方法思路

**重新定义目标参数**：不估计全局 ATE，而是仅在**重叠性良好的分组**内估计分组 ATT（Group-specific ATT），再汇报各组结论。

### 步骤

1. **按业务分组划分数据**（如：城市等级、用户活跃度、品类等）
2. **逐组检查重叠性**：
   - 对每个分组分别绘制倾向得分分布图
   - 剔除重叠性差的分组（并明确说明，不外推结论）
3. **在重叠性良好的分组内进行 PSM**：
   - 各组独立匹配，各组独立检查平衡性
4. **汇报分组 ATT**，注意结论的适用范围限制

## 注意事项

- 分组标准应**在分析前预先设定**（Pre-registration），避免事后选择性报告
- 需明确说明哪些分组被排除及原因
- 若多个分组均有结果，可加权汇总成整体估计（注意权重设计）

## 与 LATE 路径的区别

| 特征 | 分组匹配 | LATE (RDD/IV) |
|------|----------|---------------|
| 分组依据 | 业务定义的分组 | 自然断点或工具变量 |
| 估计对象 | 各业务组内 ATT | 边际人群的局部效应 |
| 是否需要工具变量 | 否 | 是（IV/RDD 路径） |
